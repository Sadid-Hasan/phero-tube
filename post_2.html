The Use Cases of Null and Undefined

Null and undefined are two important concepts in programming languages that indicates the presence or absences the value of an assigned variable.

Null: Null is often used at the time of absence, initializations in of any value of a variable. Null is used for the validation, resetting of values or use as an optional function parameter. Sometimes null is used to represent the missing result or non-existing data.

Undefined: When a variable is declared but not initialized the value is automatically turned to undefined. Sometimes for indicating missing parameter of an argument Undefined is used.

The use case of null and undefined varies from programming languages. Improper uses of these case causes bugs in the code.
