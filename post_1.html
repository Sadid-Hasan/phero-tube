Scope of Var, Let, and Const

For declaring variable in JavaScript, we can declare variable by using three different keywords. They are var, let, const. Letâ€™s see the scope among var, let & const.

Scope essentially means where these variables are available for use

Var: The scope is global when a var variable is declared outside a function. This means that any variable that is declared with var outside a function is available for use in anywhere of the code. var is function scoped when it is declared within a function. This means that it is available and can be accessed only within that function. Var is considered outdated in modern JavaScript as it has some issues related to scoping.

Let and Const: Both let and const have block-level scope. This means they are only accessible within the nearest pair of curly braces {} where they are defined. But let is mutable, which means we can reassign its value. Const is also block-scoped but is used for variables whose values should not be reassigned after initial assignment.

In modern JavaScript, it's generally recommended to use let and const for variable declarations as they provide more predictable scoping behavior compared to var.
